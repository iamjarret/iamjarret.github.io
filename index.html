July 15th: Q&A with software developer
Begin with data, fix an epsilon and construct a topological structure, called a complex. The complex has vertices, edges, faces, and generalization of these - n-simplices.

Is it a clustering algorithm?
If cluster is represented as connected components in complex then the cluster would be only a subset of information in complex's edge structure.

Complex depends on choice of epsilon, do you optimally choose one?
No, choose every one and consider the collection of all spaces each generates. This is called the persistence module. There exists a natural metric to relate two persistence modules and say how far apart the two structures are.

So like rotating the data in space and asking how dissimilar?
Yes, but with rotation whole complex structure is actually invariant under this transformation. This follows from the epsilon-balls being symmetric.

But, moving one data point slightly away will change the structure?
Yes, perfect. For some choices of epsilon the structures will still be the same, so these two data sets would be very similar and that would be captured in a small distance between persistence modules.

What are examples?
You can talk about "spaces" very abstractly. In chemistry chemists study "chemical space" the space of all possible chemicals even ones not yet discovered. These tools can describe these spaces from the data available. More concretely, a robotic arm moves in "configuration space." The shortest path in the configuration space would be an answer to how the robotic arm should accomplish a certain task. Yet, minimizing a function in a space with poor geometry could be impossible in real-time. Complexes built from the set of possible robotic arm configurations are approximations and geometrically nice.
